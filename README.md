# Demographics
Сайт для демографического прогнозирования
Сайт предназначен для демографического прогнозирования на основе данных из открытых источников. В качестве источников используются:

РЕШ ЦДИ — данные доступны и используются в текущей версии;

Росстат — ранее поддерживался, но в текущей версии недоступен, так как Росстат закрыл публичный доступ к данным.

Приложение позволяет:

выполнять демографические прогнозы по различным параметрам;

рассчитывать коэффициенты дожития на основе линейной регрессии по историческим данным;

при необходимости задавать собственные значения коэффициентов, если исторические данные недоступны или требуется модельный сценарий;

строить прогнозы для всех регионов России,

учитывать типы населённых пунктов,

фильтровать данные по полу и возрастным группам.

Таким образом, сайт предоставляет гибкую систему анализа и прогнозирования демографических показателей, позволяя проводить моделирование под различные сценарии и территориальные уровни.

Архитектура 
Архитектура проекта представляет собой веб-приложение для демографического прогнозирования, включающее клиентскую часть, backend на Django, асинхронную обработку задач и систему хранения данных.

1. Клиент (Frontend)

Пользователи: гости, зарегистрированные, администраторы.

Интерфейс: HTML/CSS/JS (Bootstrap + кастомные модули).

Используемые JS-библиотеки: Plotly.js, Lottie, noUiSlider, Feather Icons.

Клиент отправляет запросы (AJAX/HTTP) для расчёта прогноза, авторизации и получения статуса задач.

2. Веб-сервер

В продакшене работает через Nginx → Gunicorn (WSGI).

В разработке — встроенный Django Dev Server.

Отвечает за маршрутизацию и отдачу статики/медиа.

3. Django Backend

Основные приложения:

forecasting — логика прогнозирования

accounts — пользователи

data models — работа с демографическими данными

Компоненты: ORM, API views, формы, экспорт данных.

Django инициирует асинхронные задачи и читает результаты.

4. Asynchronous Tasks (Celery)

Celery-воркеры выполняют долгие задачи прогнозирования.

Redis используется как:

broker для очередей задач

cache

(опционально) хранилище результатов

Celery сохраняет прогресс и итоговые файлы прогноза.

5. File Storage

Хранение JSON-результатов прогнозов и файлов истории.

Взаимодействует с Celery и backend.

6. База данных (MySQL/MariaDB)

Хранит:

данные пользователей

исторические демографические данные

справочники

результаты прогнозов

журналы админ-панели

Django-таблицы (auth, sessions и др.)
